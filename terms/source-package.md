# source package

## Визначення (англ)

**Debian Policy Manual**  
A source package consists of a `.dsc` file, the upstream source archive, and Debian-specific patches and metadata. It is used to build one or more binary packages.

**General UNIX terminology**  
A source package contains source code, build instructions, patches, and metadata required to produce binary packages.

## Визначення (укр)

**Загальне визначення**  
Вихідний пакунок — це пакунок, що містить вихідний код, патчі, метадані та правила збирання, необхідні для створення бінарних пакунків.

**Debian / APT**  
Вихідний пакунок складається з файлів:

- `.dsc` — опис пакунка,
- `orig.tar.*` — архів вихідного коду,
- `debian.tar.*` або `diff.gz` — Debian‑специфічні зміни.

Він не встановлюється APT напряму, а використовується для побудови бінарних пакунків.

## Контекст у APT

APT може:

- завантажувати вихідні пакунки (`apt source`),
- перевіряти їхні метадані,
- використовувати їх для збирання (`dpkg-buildpackage`, `sbuild`).

Вихідний пакунок може породжувати кілька бінарних пакунків (наприклад, `bash`, `bash-doc`, `bash-builtins`).

## Можливі переклади

- **вихідний пакунок**
- пакунок вихідних кодів
- сирцевий пакунок (небажано)
- джерельний пакунок (калька)

## Аргументи за/проти

### «вихідний пакунок»

**За:**

- усталений переклад у Debian/Ubuntu;
- коротко, природно, технічно точно;
- узгоджується з протиставленням *binary vs source*;
- не містить кальок.

**Проти:**

- може асоціюватися з “output”, але контекст знімає неоднозначність.

### «пакунок вихідних кодів»

**За:**

- максимально точний.

**Проти:**

- довгий;
- не використовується в Debian‑перекладах.

## Обраний переклад

**вихідний пакунок**

## Примітки

- Вихідний пакунок — це не архів `.deb`.
- Один вихідний пакунок може породжувати багато бінарних пакунків.
