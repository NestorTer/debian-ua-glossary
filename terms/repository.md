# repository

## Визначення (англ)

**General UNIX / Linux terminology**  
A repository is a structured collection of packages, metadata, and indexes that a package manager can access to install, update, or remove software.

**Debian / APT**  
A repository is a network-accessible (or local) collection of Debian packages, organized into distributions (stable, testing, unstable), components (main, contrib, non-free), and architectures. It provides metadata files (Release, Packages, Sources) used by APT to resolve dependencies and select package versions.

## Визначення (укр)

**Загальне визначення**  
Сховище — це структурована колекція пакунків і метаданих, доступна менеджеру пакунків для встановлення, оновлення або видалення програмного забезпечення.

**Debian / APT**  
Сховище — це джерело пакунків, організоване за дистрибутивами, компонентами та архітектурами. Воно містить:

- бінарні пакунки (`.deb`);
- вихідні пакунки (`.dsc`, `orig.tar.*`, `debian.tar.*`);
- індекси (`Packages`, `Sources`);
- метадані (`Release`, `InRelease`, `Release.gpg`).

APT використовує ці дані для визначення доступних версій, залежностей і оновлень.

## Контекст у APT

APT працює зі сховищами через:

- `/etc/apt/sources.list`
- `/etc/apt/sources.list.d/*.list`

Сховище визначає:

- які пакунки доступні;
- які версії є кандидатами для встановлення;
- які компоненти дозволені (main, contrib, non-free);
- які релізи використовуються (stable, testing, unstable);
- які архітектури підтримуються.

APT отримує інформацію зі сховищ через:

- `apt update` — оновлення індексів;
- `apt install` — встановлення пакунків зі сховищ;
- `apt upgrade` — оновлення пакунків до версій зі сховищ.

## Можливі переклади

- **сховище**
- репозиторій (допустимо, але менш бажано)
- репо (жаргон)

## Аргументи за/проти

### «сховище»

**За:**

- усталений переклад у Debian, Ubuntu, KDE, GNOME;
- відповідає українській термінології;
- природно звучить у контексті APT;
- уникає кальки з англійської;
- добре поєднується з іншими термінами:  
  - сховище пакунків  
  - джерело сховища  
  - дзеркало сховища  
  - ключі сховища

**Проти:**

- новачки інколи звикли до слова “репозиторій”.

### «репозиторій»

**За:**

- звичне слово для користувачів GitHub/GitLab.

**Проти:**

- калька з англійської;
- не відповідає українській термінотворчій традиції;
- у Debian‑перекладах використовується рідко;
- створює плутанину з Git‑репозиторіями.

## Обраний переклад

**сховище**

## Примітки

- Уникаємо кальки “репозиторій пакунків” → **сховище пакунків**.
- “Репо” допускається лише як жаргон у неформальних текстах.
- Не плутати сховище (repository) з дзеркалом (mirror) — дзеркало є копією сховища.

## Зауваження щодо вживання

У неформальній розмові без контексту слово «сховище» може бути сприйняте неоднозначно.  
У таких випадках варто уточнювати: «сховище пакунків», «APT‑сховище» або «сховище Debian».  
Термін «репозиторій» використовується переважно у контексті Git і не є бажаним для APT.
