# release

## Визначення (англ)

**General software terminology**  
A release is a published, versioned state of a software project or distribution, typically associated with a specific set of packages, metadata, and stability guarantees.

**Debian / APT**  
A release is a defined state of a Debian distribution (such as stable, testing, unstable), represented by metadata files (`InRelease`, `Release`, `Release.gpg`) that describe available packages, their versions, checksums, and repository structure.

## Визначення (укр)

**Загальне визначення**  
Реліз — це опублікований, версійований стан програмного забезпечення або дистрибутива, що містить визначений набір пакунків і метаданих.

**Debian / APT**  
Реліз — це стан дистрибутива Debian (наприклад, *stable*, *testing*, *unstable*), описаний метаданими у файлах `InRelease`, `Release` та `Release.gpg`.  
Ці файли визначають:

- структуру сховища,
- доступні компоненти (main, contrib, non-free),
- контрольні суми індексів,
- версії пакунків,
- підтримувані архітектури,
- підпис GPG, що підтверджує автентичність.

## Контекст у APT

APT використовує інформацію про релізи для:

- визначення доступних версій пакунків;
- вибору candidate version;
- перевірки цілісності та автентичності сховища;
- розв’язання залежностей;
- оновлення системи.

Основні файли релізу:

- **InRelease** — метадані релізу з вбудованим GPG‑підписом;  
- **Release** — метадані релізу без підпису;  
- **Release.gpg** — окремий GPG‑підпис для файлу `Release`.

Релізи організовані за:

- **дистрибутивами**: stable, testing, unstable;  
- **кодовими назвами**: bookworm, trixie, sid;  
- **компонентами**: main, contrib, non-free;  
- **архітектурами**: amd64, arm64, i386 тощо.

## Можливі переклади

- **реліз**
- випуск (можливо, але менш уживане)
- видання (небажано)
- релізація (помилково)

## Аргументи за/проти

### «реліз»

**За:**

- усталений термін у Debian, Ubuntu, KDE, GNOME;
- відповідає міжнародній технічній термінології;
- природно поєднується з іншими термінами:  
  - реліз дистрибутива  
  - файли релізу  
  - підпис релізу  
  - релізні метадані  
- не має омонімії в технічному контексті.

**Проти:**

- іншомовне походження, але це прийнятно для технічних термінів.

### «випуск»

**За:**

- українське слово.

**Проти:**

- не усталене в Debian‑культурі;
- звучить неприродно у складених термінах;
- може плутатися з “випуск оновлень”.

## Обраний переклад

**реліз**

## Примітки

- Не плутати реліз (стан дистрибутива) з пакунком або версією пакунка.
- Файли `InRelease`, `Release` та `Release.gpg` є частиною механізму перевірки автентичності сховища.
- Кодова назва (bookworm, trixie, sid) — це не реліз сама по собі, а ідентифікатор релізу.
