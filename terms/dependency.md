# dependency

## Визначення (англ)

**Oxford Learner’s Dictionary**  
A thing that is needed or required in order for something else to function or exist.

**Merriam‑Webster**  
A condition in which one thing relies on another; something that is dependent on something else.

**Computer Science (general)**  
A relationship in which one software component requires another component to be present, installed, or functioning correctly.

## Визначення (укр)

**Загальномовне визначення**  
Залежність — це стан або відношення, коли один об’єкт потребує іншого для існування чи функціонування.

**Інформатика**  
Залежність — це вимога, за якої один компонент програмного забезпечення потребує наявності іншого компонента для коректної роботи.

**APT / Debian**  
Залежність — це інший пакунок або віртуальний пакунок, який повинен бути встановлений, налаштований або доступний, щоб даний пакунок міг працювати.  
У Debian залежності визначаються через поля керування пакунками (`Depends`, `Recommends`, `Suggests`, `Breaks`, `Conflicts`, `Provides`) відповідно до Debian Policy Manual.

## Контекст у APT

APT виконує **розв’язання залежностей** (dependency resolution), визначаючи:

- які пакунки потрібно встановити разом із основним;
- які є обов’язковими (`Depends`);
- які рекомендованими (`Recommends`);
- які бажаними (`Suggests`);
- які конфліктують (`Conflicts`);
- які замінюють інші (`Replaces`);
- які забезпечують віртуальні функції (`Provides`).

Невиконані залежності призводять до станів:

- **unmet dependencies**  
- **broken packages**  
- **held back**  
- **cannot resolve dependencies**

## Можливі переклади

- **залежність**
- вимога (непридатно в контексті APT)
- залежний компонент (надто довго)
- залежний пакунок (уточнення, а не термін)

## Аргументи за/проти

### «залежність»

**За:**

- усталений термін у програмуванні та системному адмініструванні;
- використовується в Debian, Ubuntu, KDE, GNOME;
- відповідає англійському dependency;
- природно поєднується з іншими термінами:  
  - залежності пакунка  
  - невиконані залежності  
  - розв’язання залежностей  
  - дерево залежностей  
- не має омонімії.

**Проти:**

- для новачків може звучати абстрактно, але це швидко минає.

### «вимога»

**За:**

- інколи використовується в документації.

**Проти:**

- не передає структури dependency graph;
- не узгоджується з Debian‑термінологією;
- не підходить для складених термінів (reverse dependencies, dependency chain).

## Обраний переклад

**залежність**

## Примітки

- Уникаємо кальок типу “депенденсі”.
- У складених термінах використовуємо усталені форми:  
  - dependency resolution → **розв’язання залежностей**  
  - dependency tree → **дерево залежностей**  
  - reverse dependencies → **зворотні залежності**  
  - unmet dependencies → **невиконані залежності**  
- Не плутати dependency (залежність) із requirement (вимога).
